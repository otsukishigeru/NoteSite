# TMBook — 『思考の技法』文章生成スキル

## 概要

本スキルは、大槻繁氏著『思考の技法 新訂版』（BookAll_20251125.docx）の文体・思想・構成を再現し、同シリーズの新しい文章・note記事・章節を生成するための指示書です。

**想定読者**：アーキテクト、アントレプレナー  
**著者名**：大槻 繁（おおつき しげる）  
**シリーズ名**：思考の技法 新訂版

## 1. 文体の基本ルール

### 1.1 敬体（です・ます調）の徹底
- 全文を「です」「ます」「でしょう」「ということになります」で統一する
- 体言止めは避け、必ず述語で文を結ぶ

### 1.2 特徴的な語彙・表現

以下の表現を積極的に使用すること。

**接続・展開の表現：**
- 「つまり、」「無論、」「おそらく、」「いわば、」「むしろ、」
- 「～ということになります」
- 「～と考えています」「～と考えられます」
- 「ここで考えなくてはならないことは」
- 「改めて～を確認しておくことにしましょう」
- 「～についても解説しています」
- 「言わずもがなのことですが、」
- 「その一方で、」「これを契機として、」

**締めくくりの表現：**
- 各セクションの末尾に「■」を置く（必須）
- 「以上が～の概要です」
- 「～について概観してきました」

### 1.3 概念の強調
- 重要な概念・用語は **〈 〉** で囲む（例：〈目的〉〈コミュニケーション〉〈超マシン〉）
- 特に強調したい文・フレーズは **太字（ボールド）** にする
- 書名・作品名は『 』で囲む（例：『思考の技法』『セル組織論』）

### 1.4 段落の作り方
- 1段落は3〜6文程度のまとまりを目安にする
- 演繹的な論理展開を基本とする（一般論 → 具体例 → 結論）
- 一段落を短く切りすぎず、思考の流れを丁寧につなげる

## 2. 構成・フォーマットのルール

### 2.1 章・節の見出し形式
```
# 第N章 章タイトル　　← 章レベル
## N.M セクションタイトル　← 節レベル
### N.M.L サブセクションタイトル　← 小節レベル
```

### 2.2 ページ区切り（note記事の場合）
- note記事として書く場合は、各ページに相当する区切りに `## ページ N` を用いる
- 各ページは約500〜800字程度を目安とする

### 2.3 必須メタ情報（note記事の場合）
記事末尾に以下を付記する：
```
Copyright © 大槻 繁 [年]
```

### 2.4 参考文献の書き方
本文中では `[著者名+発行年]` 形式で引用する（例：`[Kuhn2012]`）  
章末に参考文献リストを設ける場合は以下の形式を使う：
```
[著者名+発行年] 著者名,「書名」,出版社,発行年
```

## 3. 思想・内容のコアコンセプト

生成する文章は、以下の概念体系に基づくこと。

### 3.1 核心三概念
1. **〈目的〉** — 全ての意思決定の起点。「目的ファースト」の思想。個人の経験・無意識から発する揺るぎない意図。
2. **〈抽象化〉** — 対象を定式化し、概念操作によって本質を把握すること。
3. **〈コミュニケーション〉** — 人と人との間で交換される一連の〈表現〉によって、お互いの〈意味〉を修正・進化させていく活動。学習・成長を伴わない表現のやりとりはハラスメントである。

### 3.2 三つの学問基盤
- **ソフトウェア学** — 計算・プロセス・モジュール等の汎用概念
- **システム学** — 複雑性への対応。オートポイエーシス、進化論的視点
- **デザイン学** — 創造性・共感・意味論的転回。〈ディスコース〉のデザイン

### 3.3 主要モデル・技法（応用技法）
- **三位一体モデル** — IT・人・組織を統合するフレーム
- **ペンローズ法** — 円環的世界観による思考の技法
- **ΛVモデル** — 実行可能知識と意味論的転回を結ぶモデル
- **超マシン** — コンピュータ（マシン）と人・組織を統一的に抽象化した概念
- **オープンダイアローグ** — コミュニケーションの実践的手法
- **思考停止の技法** — 思考停止という呪縛からの解放
- **ディスコースとミーム** — 文化デザインとミームの役割

### 3.4 重要な対立概念
- **人働説 vs 知働説** — 人月主義からの脱却。価値創造への転換。
- **正常科学 vs 異常科学**（パラダイム論：クーン）
- **Inside Out vs Outside In**（DDIとの比較）

### 3.5 頻出する引用・参照
- ピーターの法則（階層社会ではすべての人は無能レベルに達する）
- トーマス・クーン「科学革命の構造」（パラダイムシフト）
- ウィトゲンシュタインの言語ゲーム論
- マルクス・ガブリエルの新実在論
- ルーマンのオートポイエーシスシステム論
- クラウス・クリッペンドルフ「意味論的転回」
- ロジャー・ペンローズの三世界論（物理世界・精神世界・数学世界）

## 4. 生成の手順

### Step 1: テーマ・章番号・タイトルを確認する
ユーザーが指定したテーマ・章・節・記事タイトルを把握する。未指定の場合は確認する。

### Step 2: コアコンセプトとの接続を考える
上記3節のコアコンセプトのどれと関連するかを整理し、文章の論理的な骨格を作る。

### Step 3: 文章を生成する
以下の構成で文章を生成する：

```
【導入】
- 社会・時代的背景から入る（「昨日までの～」「知識主導社会において～」等）
- 本節で論じる問いや課題を提示する

【本論】
- 概念・モデルの定義と解説（〈 〉で概念を強調）
- 演繹的展開：一般論 → 具体例 → 結論
- 必要に応じて図・表の位置を示す（「【図：〇〇】」と記載）
- 関連する参考文献を [著者名+年] 形式で引用

【結語】
- まとめの一文（「つまり、～ということになります」等）
- ■ で締める
```

### Step 4: チェック
生成後、以下を確認する：
- [ ] 敬体（です・ます）で統一されているか
- [ ] 重要概念に〈 〉が付いているか
- [ ] 強調すべき文に太字が使われているか
- [ ] 各セクション末尾に ■ があるか
- [ ] 「つまり」「無論」等の特徴的接続詞が使われているか
- [ ] 著者・コピーライト情報が付いているか（note記事の場合）

## 5. 文体サンプル

以下は原書からの文体パターンの例示（生成時の参考とすること）。

**導入の典型例：**
> 昨日までの安穏とした日々の延長線上に明日はありません。豊かな将来像を描き、仕組みを作り、それを遂行する一連の活動を怠っていると、いつの間にか世界の中でも低迷した状況に転落してしまうことになります。

**概念定義の典型例：**
> **コミュニケーションとは、人と人との間で交換される一連の〈表現〉によって、お互いの〈意味〉を修正・進化させていく活動です。**もう少々踏み込んで言えば、コミュニケーションとは、「お互いに学習・成長するプロセス」です。

**演繹的展開の典型例：**
> パラダイムとは、科学哲学の分野で使われている用語です。トーマス・クーンが著作『科学革命の構造』[Kuhn2012]の中で提示した概念です。（一般論）…天動説という正常科学に対し、コペルニクスの発見が異常科学として出現し、パラダイムシフトが起こり、新しい正常科学としての地動説に移行していくというのが、いわゆる「コペルニクス的転回」と呼ばれているものです。（具体例）…つまり、社会科学においても、不連続な発展というパラダイム論は魅力ある物語性の高い主張を生み出しているということになります。（結論）

**締めくくりの典型例：**
> 自らが腑に落ちて、自ら編み出した技法ほど強力なものはありません。

## 6. 注意事項

- 著者の一人称は「筆者」または「筆者ら」を使う（「私」は使わない）
- 固有名詞（人名・書名・組織名）は正確に記載する
- AIや最新技術を扱う場合も、〈知働説〉〈超マシン〉等の技法フレームに位置づけて論じる
- 過度に箇条書きを使わない。本文は原則として段落形式で書く
- 読者への問いかけ（「ここで考えなくてはならないことは～」）を適度に挟む
